**And:** [Co-Maintainer Name] ("Co-Maintainer")  
**Project:** Nexus Bot

## 1. Role and Responsibilities

The Co-Maintainer agrees to:
- Help maintain and improve the Nexus Bot codebase
- Review pull requests and manage issues
- Assist with bot deployment and monitoring
- Provide support to the community
- Act professionally and in the project's best interests

## 2. Access and Permissions

The Co-Maintainer will receive:
- GitHub repository write access
- Discord support server moderator role
- Dashboard access (level determined by Lead Maintainer)
- Access to internal discussions and planning channels

## 3. Confidentiality Agreement

### 3.1 What is Confidential

The Co-Maintainer agrees that the following information is **CONFIDENTIAL** and must not be shared publicly or with third parties:

- **Internal Metrics:** Server counts, user statistics, growth rates, retention data, analytics
- **Strategic Plans:** Marketing strategies, growth tactics, partnership discussions, business plans
- **Competitor Research:** Intelligence on competing bots (Wick, etc.), their weaknesses, pricing analysis
- **Financial Information:** Revenue, costs, donations, monetization plans, expenses
- **Security Issues:** Unpatched vulnerabilities, exploit details, security incidents (before public disclosure)
- **User Data:** Private user information, server data, messages, moderation logs, ban reasons
- **Access Credentials:** API keys, bot tokens, passwords, .env file contents, server access details
- **Pre-Release Features:** Unreleased features, upcoming announcements, beta tests (before public reveal)
- **Internal Discussions:** Private conversations in maintainer channels, strategy discussions, debates

### 3.2 What is NOT Confidential

The following information is **PUBLIC** and can be freely discussed:
- All source code in the public GitHub repository
- Public documentation, guides, and tutorials
- Public feature requests and bug reports on GitHub
- General statements like "we're working on improving X feature"
- Public announcements made by the Lead Maintainer

### 3.3 Duration

This confidentiality obligation:
- Begins on the date of this agreement
- Continues during the Co-Maintainer's active role
- **Remains in effect for 2 years after leaving the project**

### 3.4 Permitted Disclosures

Confidential information may be disclosed:
- With explicit written permission from the Lead Maintainer
- When required by law (with advance notice to Lead Maintainer if possible)
- To resolve critical security vulnerabilities (coordinated disclosure only)

## 4. Intellectual Property

### 4.1 Existing IP
- The Lead Maintainer retains ownership of the Nexus Bot project and brand
- All existing code, trademarks, and assets remain with the Lead Maintainer

### 4.2 Contributions
- Code contributions are made under the project's MIT License
- Non-code contributions (ideas, strategies, docs) are jointly owned
- The Lead Maintainer retains final decision-making authority

### 4.3 Restrictions
The Co-Maintainer agrees NOT to:
- Create competing bots using confidential information
- Fork Nexus with confidential strategies/data for 2 years after leaving
- Sell or license confidential information to competitors
- Use confidential information for personal profit

## 5. Termination

### 5.1 Voluntary Departure
- Either party can end this agreement with 2 weeks notice
- Co-Maintainer must return all credentials and access
- Confidentiality obligations continue for 2 years

### 5.2 Termination for Cause
The Lead Maintainer may immediately terminate for:
- Breach of confidentiality
- Malicious actions (sabotage, data theft, etc.)
- Serious misconduct or unprofessional behavior
- Inactivity for 90+ days without communication

### 5.3 Upon Termination
The Co-Maintainer must:
- Delete all local copies of confidential information
- Destroy all credentials (.env files, API keys, etc.)
- Stop using any Nexus-related access immediately
- Transfer any project assets back to Lead Maintainer

## 6. Data Protection

The Co-Maintainer agrees to:
- Handle user data in compliance with Discord ToS and applicable laws
- Use user data ONLY for bot operation and moderation
- Never sell, share, or misuse user data
- Report any data breaches immediately
- Follow security best practices (2FA, secure passwords, encrypted storage)

## 7. Dispute Resolution

### 7.1 Good Faith
Both parties agree to resolve disputes through good-faith discussion first.

### 7.2 Mediation
If discussion fails, disputes will be resolved through mediation before legal action.

### 7.3 Governing Law
This agreement is governed by the laws of [YOUR COUNTRY/STATE].

## 8. Miscellaneous

### 8.1 No Employment
This agreement does not create an employment relationship. The Co-Maintainer is a volunteer contributor.

### 8.2 Entire Agreement
This document represents the complete agreement between parties regarding co-maintainership.

### 8.3 Amendments
This agreement can only be modified with written consent from both parties.

### 8.4 Severability
If any provision is found invalid, the rest of the agreement remains in effect.
